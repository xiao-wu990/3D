# P2P 文件传输 - GitHub Pages 版本

基于 **PeerJS** 的点对点文件传输工具，可以部署到 GitHub Pages，完全免费使用！

## ✨ 特性

- ✅ **完全免费** - 部署到 GitHub Pages 零成本
- ✅ **无需服务器** - 纯前端 + PeerJS 公共服务器
- ✅ **P2P 传输** - 文件直接在两设备间传输，不经过中间服务器
- ✅ **多人连接** - 支持多设备同时连接
- ✅ **文字聊天** - 实时聊天功能
- ✅ **麦克风喊话** - 实时语音传输
- ✅ **文件在线预览** - 支持图片、视频、PDF、文本等预览
- ✅ **语音/视频通话** - 支持一对一和多人通话
- ✅ **用户昵称** - 个性化标识
- ✅ **深色模式** - 支持主题切换
- ✅ **永久在线** - GitHub Pages 24/7 可用
- ✅ **任意设备** - 支持浏览器、手机、电视等

## 🚀 快速开始

### 本地测试

直接用浏览器打开 `index.html` 文件即可测试。

### 部署到 GitHub Pages

#### 步骤 1：准备代码

```bash
# 1. 进入项目目录
cd C:\Users\Lenovo\CodeBuddy\20260228200833

# 2. 初始化 Git（如果还没有）
git init

# 3. 添加文件
git add index.html README-GITHUB.md

# 4. 提交
git commit -m "Add P2P file transfer"

# 5. 推送到 GitHub
git remote add origin https://gitee.com/x-ci-chuan/wjcs.git
git push -u origin main
```

#### 步骤 2：启用 GitHub Pages

1. 访问你的 GitHub 仓库
2. 进入 **Settings** → **Pages**
3. 在 **Source** 中选择：
   - Branch: `main`
   - Folder: `/ (root)`
4. 点击 **Save**

#### 步骤 3：获取访问地址

等待 1-2 分钟，访问：
```
https://你的用户名.github.io/你的仓库名/
```

**示例：**
- GitHub 用户名：`xiao-wu990`
- 仓库名：`wjcs`
- 访问地址：`https://xiao-wu990.github.io/wjcs/`

## 📱 使用方法

### 1. 设置昵称
- 打开页面后,输入你的昵称
- 点击确定保存
- 昵称会显示在发送的消息和文件中

### 2. 选择模式

**发送端**
1. 点击 **"发送端"**
2. 等待生成 ID
3. 复制你的 ID，分享给接收端

**接收端**
1. 点击 **"接收端"**
2. 等待生成 ID
3. 复制 ID，分享给发送端

**多人模式**
1. 点击 **"多人模式"**
2. 可以同时连接多个设备
3. 使用 **"+ 添加连接"** 按钮添加更多设备

### 3. 建立连接
1. 输入对方 ID 并点击 **"连接"**
2. 等待连接成功

### 4. 开始使用

**文件传输**
- 拖拽文件到上传区域
- 或点击选择文件
- 支持多人同时发送/接收
- 可预览的文件会显示预览按钮

**文字聊天**
- 在聊天框输入消息
- 按回车或点击发送
- 实时显示所有聊天记录

**麦克风喊话**
- 点击 **"开启麦克风"**
- 你的语音会实时传送到所有连接的设备
- 可以随时静音/取消静音

**语音/视频通话**
- 点击 **"语音通话"** 或 **"视频通话"**
- 对方收到提示,选择接听或拒绝
- 通话中可以随时挂断
- 多人模式下可以邀请其他人加入

## ⚙️ 工作原理

### PeerJS

使用 **PeerJS** 库实现 WebRTC P2P 连接：

- PeerJS 提供免费的公共信令服务器
- 通过信令服务器交换连接信息
- 建立 WebRTC 直接连接
- 文件在两设备间直接传输

### 优势

- **无需自建服务器**
- **GitHub Pages 完全支持**
- **传输速度快**（P2P 直连）
- **完全免费**

## ⚠️ 注意事项

### 1. PeerJS 公共服务器限制

- 免费版有连接数限制
- 可能有速率限制
- 大文件传输可能不稳定

### 2. 文件大小限制

- 浏览器有内存限制
- 建议单个文件不超过 100MB
- 大文件建议分段传输

### 3. 网络要求

- 支持 WebRTC 的浏览器
- UDP 端口开放（部分网络可能受限）
- NAT 穿透（部分网络可能失败）

### 4. 兼容性

**支持的浏览器：**
- ✅ Chrome/Edge（推荐）
- ✅ Firefox
- ✅ Safari（部分支持）
- ❌ 电视浏览器（大多数不支持 WebRTC）

**不支持的设备：**
- ❌ 老旧浏览器
- ❌ 部分电视浏览器
- ❌ 不支持 WebRTC 的环境

## 🔧 故障排除

### 问题 1：无法连接

**解决方案：**
- 检查网络连接
- 确认 ID 正确
- 尝试重新连接
- 检查浏览器是否支持 WebRTC

### 问题 2：文件传输失败

**解决方案：**
- 检查文件大小（建议 < 100MB）
- 检查网络稳定性
- 尝试重新发送

### 问题 3：电视无法使用

**解决方案：**
- 大多数电视浏览器不支持 WebRTC
- 建议使用手机/电脑
- 或安装支持 WebRTC 的应用

### 问题 4：GitHub Pages 访问慢

**解决方案：**
- GitHub Pages 在国内访问较慢
- 考虑使用 Gitee Pages（国内更快）
- 或使用 CDN 加速

## 🌐 其他部署平台

2. 创建仓库并上传 `index.html`
3. 进入仓库 **服务** → **Gitee Pages**
4. 启用 Pages 服务
5. 获取访问地址

1. 注册 Netlify：https://app.netlify.com
2. 拖拽 `index.html` 到 Netlify
3. 自动部署完成

### Vercel（全球 CDN）

1. 注册 Vercel：https://vercel.com
2. 导入 GitHub 仓库
3. 自动部署完成
